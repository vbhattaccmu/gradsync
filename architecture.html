Architecture

gradsync layers systems code (Rust) over an ergonomic Python shell. Nothing is torch-specific.

How it works

1. Rendezvous

Rank 0 creates a 128-byte NCCL unique id and broadcasts it over TCP to every other rank. After this handshake, NCCL builds its own transports.

2. Communicator init

Every rank calls ncclCommInitRank. NCCL automatically probes the hardware fabric and picks the fastest path:

  • Inside a node: NVLink / NVSwitch
  • Across nodes: Network (GPUDirect-TCPX / RoCE / InfiniBand)

gradsync doesn't choose the transport — it reports which one was chosen.

3. Broadcast weights

DDP broadcasts rank 0's initial parameters so all ranks start with identical weights.

4. Overlapped AllReduce

Parameters are grouped into ~25 MB buckets in reverse order. An autograd hook fires when each parameter's gradient is ready. When a bucket completes, its fused in-place average-AllReduce is enqueued on a dedicated comm stream during backward, overlapping with compute still running on the default stream. No host copies — the reduction is GPU-to-GPU.

Stream ordering (why it's correct)

The core's collectives are enqueue-only — they never synchronize. Ordering is the caller's job, done with CUDA stream waits so the two streams interleave safely:

backward (compute):
... grad_k ready ──┐
│ comm_stream.wait_stream(compute)
comm stream:
AllReduce ─────────┘ (runs concurrently)
sync_gradients():
compute.wait_stream(comm_stream)
optimizer.step():
──► safe: grads fully reduced
↓ Compute waits for comm
comm_stream.wait_stream(compute) guarantees AllReduce never starts before gradient kernels finish.
↓ Optimizer waits for comm
compute.wait_stream(comm_stream) guarantees weights never update mid-reduction.

Build modes

Mode Command Collectives? Location
Stub cargo build No (error) Laptop (no GPU)
Real maturin build --features nccl Yes GPU host (CUDA + NCCL)
Why stub mode? It lets you develop rendezvous, topology parsing, and the Python SDK without a GPU. Collectives light up when you build on a GPU host with CUDA + NCCL.

Why we don't emulate NVLink

NVLink and NVSwitch are physical interconnect hardware. You get them by renting a multi-GPU box (A100/H100). gradsync's honest claim is:

  • It drives real NCCL collectives
  • Uses NVLink automatically when hardware has it
  • Uses the network across nodes
  • report_topology() and NCCL_DEBUG=INFO prove which path each collective took

We don't pretend to simulate hardware we can't control.
